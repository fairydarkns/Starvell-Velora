# StarvellVelora

StarvellVelora — Telegram-бот для продавцов Starvell. Проект объединяет мониторинг заказов, обработку чатов, автовыдачу, шаблоны ответов, систему плагинов и seller-side действия в одном рабочем контуре.

## Возможности

- Отслеживание новых заказов и новых сообщений покупателей.
- Ответы в чаты Starvell прямо из Telegram.
- Подтверждение и завершение заказов со стороны продавца.
- Автовыдача, автовосстановление, автоподнятие, автоответ, черный список и уведомления.
- Раздельные прокси для Starvell и Telegram.
- Подключаемые плагины без правки ядра.
- Первичная настройка через терминал с последующим обычным запуском.

## Преимущества

- Один хаб управления в Telegram вместо ручной работы через браузер.
- Четкое разделение на конфиг, логи, хранилище и плагины.
- Только ручное обновление с backup перед заменой файлов.
- Гибкая система расширений для интеграций и кастомной логики.
- Нормально подходит для постоянной работы на сервере.

## Что есть внутри

### Starvell

- Разбор заказов и получение деталей заказа.
- Опрос чатов и обработка входящих сообщений.
- Seller-side complete / confirm.
- Автоподнятие и автовосстановление.
- Заготовки ответов и пользовательские команды.

### Telegram

- Панель управления основными настройками.
- Настройка уведомлений и доступа.
- Быстрые ответы, черные списки, пользовательские команды, автоответ.
- Сервисные уведомления и доступ к логам.

### Плагины

- Плагины подключаются из папки `plugins/`.
- Можно добавлять свои команды, callback-кнопки, текстовые обработчики и хуки на события.
- Встроенная интеграция SteamGifts работает через тот же слой расширений.

## Первый запуск

При первом запуске откроется мастер настройки в терминале.

Он запрашивает:

- токен Telegram-бота
- пароль доступа к Telegram-панели
- `session_cookie` Starvell
- прокси для Starvell при необходимости
- прокси для Telegram при необходимости
- базовые параметры запуска

Основной конфиг хранится в `configs/_main.cfg`.

Рабочие данные лежат в `storage/`.

Логи пишутся в `logs/log.log`.

## Установка

### Windows

1. Установите Python 3.11+.
2. Откройте папку проекта.
3. Запустите `Setup.bat`.
4. После установки зависимостей запустите `Start.bat`.

### Linux

1. Убедитесь, что установлены `python3` и `python3-venv`.
2. Откройте папку проекта.
3. Выполните:

```bash
chmod +x setup.sh start.sh
./setup.sh
./start.sh
```

### Ручной запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

## Как делать плагины

Плагины загружаются из папки `plugins/`.

Рекомендуемые форматы:

- один файл: `plugins/my_extension.py`
- отдельная папка: `plugins/MyExtension/module.py`
- пакет: `plugins/MyExtension/__init__.py`

Обязательные метаданные:

```python
NAME = "My Extension"
VERSION = "1.0.0"
DESCRIPTION = "Custom behavior for StarvellVelora"
AUTHOR = "@yourname"
UUID = "11111111-2222-4333-8444-555555555555"
```

Базовые хуки:

```python
async def on_init(bot, starvell, db, extension_hub):
    ...

async def on_new_order(order_data, starvell_service=None, *args, **kwargs):
    ...

async def on_new_message(message_data, starvell_service=None, *args, **kwargs):
    ...

BIND_TO_INIT = [on_init]
BIND_TO_NEW_ORDER = [on_new_order]
BIND_TO_NEW_MESSAGE = [on_new_message]
```

Дополнительно можно использовать:

- `COMMANDS`
- `CALLBACKS`
- `TEXT_HANDLERS`
- `BIND_TO_START`
- `BIND_TO_STOP`
- `BIND_TO_DELETE`

Важно:

- не ставьте непроверенные плагины;
- плагины получают доступ к рантайму и локальным файлам;
- данные плагина лучше хранить в `storage/plugins/` или в его собственной подпапке.

## Важное

- автообновление при старте отключено;
- `/update` работает только вручную;
- перед обновлением всегда создается backup;
- хранится только 3 последних backup;
- шаблоны и пользовательские команды лежат в `storage/telegram/`.
